Hot and Cold Observables in RxJS

A cold observable creates a new execution for each subscriber ‚Äî like an HTTP call. A hot observable shares the same data stream among all subscribers ‚Äî like fromEvent or a Subject. In Angular, cold observables are common for APIs, but we can convert them to hot ones using shareReplay when caching is needed.

  • Starts emitting only when someone subscribes.
  • Each subscriber gets separate data streams.
  • Each subscriber gets a different random value.
  • Subscribers share the same source. It emits even if no one is listening yet.
  • All subscribers share the same data source.
